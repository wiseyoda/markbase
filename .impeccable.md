## Design Context

### Users
CEO (Patrick) and team collaborators (like Bill) at Level Agency. They use markbase to browse, render, share, and comment on markdown documents across GitHub repos. Primary context: reviewing strategy docs, plans, and knowledge base articles. Shared docs may be viewed by external collaborators who have never seen the app before.

### Brand Personality
Professional, clean, sharp. The interface should feel like a competent business tool — not playful, not corporate, not developer-raw. Think "senior designer's personal tool" rather than "enterprise SaaS."

### Aesthetic Direction
- **Visual tone**: Typography-first, minimal chrome, content is king. Reference: Bear, iA Writer.
- **Target emotion**: "This is easy to use" — clarity and simplicity, zero friction.
- **Theme**: Light + dark mode with manual toggle (+ system default).
- **Color**: Level Agency brand — Blue #86D5F4 primary accent. One color family only. Green #8EE34D for inline code. Pink/Orange reserved for rare accents.
- **Fonts**: Geist Sans (body), Geist Mono (code). Chosen over brand fonts for readability.
- **Anti-references**: Rainbow color schemes, cluttered dashboards, developer-raw UIs.

### Design Principles
1. **Content over chrome** — The markdown document is the star. Everything else should recede.
2. **One accent family** — Blue hierarchy for interactive/heading elements. No competing colors.
3. **Clarity at every step** — New users should immediately understand what to do next.
4. **Professional restraint** — No decorative elements that don't communicate meaning.
5. **Responsive by design** — Mobile and tablet are first-class citizens, not afterthoughts.
